Setup Guide

Setting up your Anthropic (Claude) API key

This guide walks you through every click: creating an account, adding credits, getting your API key, and connecting it to the Job Alert Scorer. Total time: about 10 minutes.

← Back to Getting Started

What you're setting up

The tool scores each job by sending that job's description and your candidate profile to Claude. To do that it needs an API key — a secret string tied to your own Anthropic account, billed pay-as-you-go.

You'll end up with a key that starts with sk-ant-, saved in a .env file in your private config folder. The tool reads it from there; it never lives in the public repo.

Step by step

Create an Anthropic account

  1. Go to console.anthropic.com
  2. Click Sign Up (or Log In if you already have an account)
  3. You can sign up with your Google account or with an email and password
  4. Verify your email if prompted
This is a separate account from Claude.ai (the chat interface). The API console is where you manage your key and billing. You can use the same email for both, but they're different logins.

Add credits (pay-as-you-go)

The API is pay-as-you-go — you load credits and they're drawn down as the tool scores jobs. There's no monthly subscription.

  1. Once logged in, click Billing in the left sidebar
  2. Click Add Payment Method and enter a card
  3. Click Add Credits and choose an amount

How much to start with: $5 is plenty. At roughly 1–2 cents per job scored, $5 covers a few hundred jobs. You can always add more later. If you're offered free trial credits for a new account, use them.

Set a spend limit. In the same billing area you can set a monthly usage limit as a safety net, so a runaway run can never cost more than you intend.

Create your API key

  1. Click API Keys in the left sidebar
  2. Click Create Key
  3. Give it a name you'll recognise, like job-alert-scorer
  4. Copy the key immediately. It starts with sk-ant- and you'll only see it once. If you lose it, just create a new one.
Treat this key like a password. Don't paste it into any public document, don't commit it to GitHub, and don't share it. The tool stores it in your private config folder, which is kept out of the repo by .gitignore.

Save your key

The tool reads your key from the ANTHROPIC_API_KEY environment variable. The simplest way to set it is a .env file in the same folder as your config.yaml. Copy the bundled .env.example and fill in your key:

# in your private config folder, next to config.yaml cp .env.example .env # then edit .env so it reads: ANTHROPIC_API_KEY=sk-ant-...

Replace sk-ant-... with the actual key you copied. That's it — the tool picks it up automatically on the next run.

Prefer to set it in your shell instead of a file? That works too:

# macOS / Linux export ANTHROPIC_API_KEY=sk-ant-... # Windows (Command Prompt) set ANTHROPIC_API_KEY=sk-ant-...
The key is only ever read from the environment (directly, or via the .env file). It does not go in config.yaml — keep your .env in your private config folder, never in the repo.

Confirm it works

Do a first scored run over a short window so any key problem surfaces immediately at almost no cost:

# scores only the last day's alerts — a few cents at most python src/main.py --config ./config.yaml --since 1d --score

If the key is set correctly, the tool fetches and scores the jobs it found. If it isn't, you'll get a clear ANTHROPIC_API_KEY is not set message or an authentication error — in which case, check:

  • Is the key copied in full, with no extra spaces?
  • Is .env in the same folder as the config.yaml you passed to --config?
  • If you used export/set, is it the same terminal session you're running the tool from?
  • Have you added credits? A key with no credits authenticates but fails when scoring.
Want to look before you spend anything at all? Drop the --score flag — python src/main.py --config ./config.yaml --since 7d just lists the jobs it finds, with no API calls and no cost.

Choose your model

The default model is claude-sonnet-4-6 — the best balance of quality and cost for scoring jobs. Set it in config.yaml:

scoring: provider: anthropic model: claude-sonnet-4-6 # recommended — best quality-to-cost ratio

Other Anthropic models you could use:

  • claude-haiku-4-5 — faster and cheaper, slightly less detailed assessments
  • claude-opus-4-8 — highest quality but more expensive; only worth it if you want the most thorough reasoning

For most people, Sonnet is the right choice. Start there and only change if you have a specific reason.

What it costs

Gmail is free. You only pay for the AI scoring, and the title pre-filter keeps most jobs from ever reaching the API. List prices per million tokens, with a rough per-job estimate (each job is a small request, and with Anthropic the rubric + profile sit in the cached part of the prompt):

ModelPrice (in / out per 1M)~ Per job
Claude Sonnet claude-sonnet-4-6$3 / $15~1–2 cents
Claude Haiku claude-haiku-4-5$1 / $5under a cent
Claude Opus claude-opus-4-8$5 / $25a few cents

A typical daily run on Sonnet is a few cents to a couple of dollars a month. Your actual cost depends on how many alerts you get and how aggressive your title filter is. Set a monthly spend limit in the console as a safety net.

Troubleshooting

"Invalid API key" / authentication error — make sure you copied the full key with no surrounding spaces, that .env sits next to your config.yaml, and (if you used export/set) that it's the same terminal session. If in doubt, create a fresh key in the console.

"Insufficient credits" — add credits in the console's billing area, and check your usage in case you've hit a spend limit you set.

Rate limited — the tool already pauses between scoring calls, but a very large batch can still hit Anthropic's limits. Wait a few minutes and run again; scoring is cached, so it picks up where it left off without re-charging you for jobs already done.

Key security. If you ever accidentally commit a key to a public repo, delete it in the console immediately and create a new one. (Anthropic may also email you if it detects a key published publicly.)
