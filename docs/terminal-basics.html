Quick Guide

Using the Terminal (you've got this)

If you've never typed a command before, the Terminal looks intimidating. It isn't. To run this tool you only ever need a couple of lines — this page is the whole routine, so you can come back and run it any time without re-reading the setup.

What the Terminal actually is

It's just a place where you type an instruction and press Enter, instead of clicking buttons.

That's the whole idea. A window where you type a line, hit Enter, and the computer does the thing. You can't break your computer by running the commands on this page, and nothing happens until you press Enter — so there's room to be careful. If a line ever looks wrong, just delete it and start again.

You'll usually hear it called the Terminal on a Mac and the Command Prompt on Windows. Same idea, slightly different name.

Opening it

On a Mac

Press Cmd + Space to open Spotlight search, type Terminal, and press Enter. A small window opens with a blinking cursor. That's it.

On Windows

Click the Start menu, type cmd, and press Enter (or click "Command Prompt"). A black window opens with a blinking cursor.

The blinking cursor means it's ready. It's waiting for you to type a line and press Enter. Nothing is happening until you do.

The only two ideas you need

Everything you do here is one of two things:

  1. Tell it which folder you're working in — using a command called cd (short for "change directory," i.e. change folder).
  2. Type a command and press Enter — to actually run the tool.

That's genuinely it. The next section is the exact routine.

Running the scorer, start to finish

This is the routine you'll repeat whenever you want to check for new jobs. Open the Terminal, then do these three things in order.

Go to the tool's folder

Type cd, then a space, then the path to the folder you unzipped. Don't know the path? You don't have to type it. Type cd and a space, then drag the folder from Finder (Mac) or File Explorer (Windows) straight into the Terminal window — it fills in the path for you. Then press Enter.

cd (drag your job-alert-scorer folder here, then press Enter)

Turn the tool on (activate its environment)

During setup, the tool was given its own little sandbox so it doesn't interfere with anything else on your computer. You switch it on with one line — different on Mac vs Windows:

On a Mac

source .venv/bin/activate

On Windows

.venv\Scripts\activate

You'll know it worked when (.venv) appears at the start of the line. (If you get a "no such file" error here, the sandbox wasn't created during setup — see the first hiccup below.)

Run it

Paste this line and press Enter. It checks for new alerts, scores them, and emails you the ranked summary:

python src/main.py --config ./config.yaml --score --email

Keeping your profile and key in a separate private folder? Point at that config instead:

python src/main.py --config ~/my-job-scorer-config/config.yaml --score --email

Want to look before spending anything? Drop --score and it just lists what it found, with no AI calls and no cost.

Tip: you can paste. Copy any command from this page, then in the Terminal press Cmd+V (Mac) or right-click → Paste (Windows). No need to type it by hand.

Your reusable cheat sheet

Fill in your own folder once and keep this somewhere handy (a sticky note, a notes app). Next time, it's just copy-paste-Enter, three lines:

# 1. go to the folder (or drag it in after "cd ") cd /path/to/your/job-alert-scorer # 2. switch the tool on (Windows: .venv\Scripts\activate) source .venv/bin/activate # 3. run it python src/main.py --config ./config.yaml --score --email

That's the entire thing you'll do each time. The first setup is the slow part; this is the easy part you'll actually repeat.

Things that look scary but are normal

  • It pauses and seems frozen. Scoring talks to the AI and takes a few seconds per job. As long as new lines keep appearing (scoring 3/12 ...), it's working — don't close the window.
  • You typed your password and nothing showed up. If a step ever asks for a password, the Terminal hides what you type on purpose — no dots, no stars. Just type it and press Enter.
  • A wall of text scrolls by. That's normal output, not errors. Errors usually say Error or Traceback explicitly.
  • You stopped it partway. No harm done — it saves each score as it goes. Run it again and it skips everything already scored, so you never wait (or pay) twice.

Common hiccups (and the fix)

"no such file or directory: .venv/bin/activate" — the tool's sandbox wasn't set up, or you're in the wrong folder. Make sure step 1 pointed at the unzipped job-alert-scorer folder. If it still happens, the one-time setup didn't finish — see the Getting Started page, or have whoever helped you set it up re-run the install step.

"command not found: python" — try python3 instead of python (Mac and Linux often use that name). If neither works, Python isn't installed yet — the Getting Started page covers that.

"can't open file 'src/main.py'" — you're not in the tool's folder. Redo step 1 (cd into the unzipped folder), then try again.

"ANTHROPIC_API_KEY is not set" — your API key isn't where the tool expects it. The Anthropic API setup guide walks through fixing that in two minutes.
